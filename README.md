# 🤖 AI Транскрибатор - Telegram Бот

**Интеллектуальный Telegram-бот для преобразования изображений с текстом в аудиофайлы с помощью технологии Whisper**

## 📋 Описание проекта

AI Транскрибатор - это современный Telegram-бот, который использует передовые технологии искусственного интеллекта для двунаправленной обработки контента: распознавания текста на изображениях и преобразования его в аудио, а также распознавания речи в голосовых сообщениях и преобразования в текст. Бот идеально подходит для создания аудиоверсий документов, транскрибации аудио, изучения языков и улучшения доступности контента.

### 🎯 Основные возможности

- 📸 **Распознавание текста** - Извлечение текста из изображений с помощью OCR
- 🎙️ **Синтез речи** - Преобразование текста в качественное аудио с помощью Whisper
- 🎤 **Распознавание речи** - Преобразование голосовых сообщений в текст
- 🔄 **Двунаправленная транскрибация** - Обработка как изображений, так и аудио
- 🌍 **Мультиязычность** - Поддержка русского и английского языков
- ⚡ **Быстрая обработка** - Оптимизированная обработка изображений и аудио
- 📱 **Удобный интерфейс** - Интуитивно понятное управление через Telegram
- 👥 **Пользовательские роли** - Разграничение доступа (Пользователь/Администратор)
- 🎛️ **Административная панель** - Веб-интерфейс для управления системой
- 📊 **Статистика и аналитика** - Детальная информация об использовании
- 🔒 **Безопасность** - Управление доступом и блокировка пользователей

## 🚀 Установка и настройка

### Предварительные требования

#### Системные зависимости
1. **Python 3.9 или выше**
2. **Tesseract OCR**
   ```bash
   # Ubuntu/Debian
   sudo apt-get install tesseract-ocr tesseract-ocr-rus tesseract-ocr-eng
   
   # CentOS/RHEL/Fedora
   sudo dnf install tesseract tesseract-langpack-rus
   # или
   sudo yum install tesseract tesseract-langpack-rus
   
   # Arch Linux
   sudo pacman -S tesseract tesseract-data-rus
   ```
3. **FFmpeg** (для обработки аудио)
   ```bash
   # Ubuntu/Debian
   sudo apt-get install ffmpeg
   
   # CentOS/RHEL/Fedora
   sudo dnf install ffmpeg
   # или
   sudo yum install ffmpeg
   
   # Arch Linux
   sudo pacman -S ffmpeg
   ```
4. **PostgreSQL** (для базы данных)
   ```bash
   # Ubuntu/Debian
   sudo apt-get install postgresql postgresql-contrib
   
   # CentOS/RHEL/Fedora
   sudo dnf install postgresql-server postgresql-contrib
   # или
   sudo yum install postgresql-server postgresql-contrib
   
   # Arch Linux
   sudo pacman -S postgresql
   
   # Docker (рекомендуется)
   docker run --name postgres -e POSTGRES_PASSWORD=password -p 5432:5432 -d postgres:15
   ```

### Установка проекта

#### Вариант 1: Локальная установка
1. **Клонируйте репозиторий**
   ```bash
   git clone <repository-url>
   cd telegram_bot
   ```

2. **Создайте виртуальное окружение**
   ```bash
   python3 -m venv venv
   source venv/bin/activate
   ```

3. **Установите зависимости**
   ```bash
   pip install -r requirements.txt
   ```

4. **Настройте базу данных**
   ```bash
   # Создание БД в PostgreSQL
   createdb ai_transcriber
   
   # Применение миграций
   alembic upgrade head
   ```

5. **Создайте файл конфигурации**
   ```bash
   cp .env.example .env
   ```

6. **Настройте переменные окружения**
   - Откройте файл `.env`
   - Добавьте ваш Telegram Bot Token
   - Настройте подключение к базе данных
   - Укажите ID администраторов

#### Вариант 2: Docker установка (рекомендуется)
1. **Клонируйте репозиторий**
   ```bash
   git clone <repository-url>
   cd telegram_bot
   ```

2. **Настройте переменные окружения**
   ```bash
   cp .env.example .env
   # Отредактируйте .env файл
   ```

3. **Запустите все сервисы**
   ```bash
   docker-compose up -d
   ```

4. **Примените миграции**
   ```bash
   docker-compose exec telegram-bot alembic upgrade head
   ```

- `/users` - Список пользователей

### Как использовать

#### 📸 Преобразование изображения в аудио
1. **Запустите бота** командой `/start`
2. **Отправьте изображение** с текстом (JPG, PNG, WEBP)
3. **Дождитесь обработки** - бот покажет прогресс
4. **Получите аудиофайл** с озвученным текстом

#### 🎤 Преобразование аудио в текст
1. **Запустите бота** командой `/start`
2. **Отправьте голосовое сообщение** через Telegram
3. **Дождитесь обработки** - бот распознает речь
4. **Получите текст** с транскрипцией речи

#### Для изображений:
- Используйте четкий, контрастный текст

- Текст должен быть горизонтальным и читаемым

#### Для голосовых сообщений:

- Избегайте сильного фонового шума
- Используйте встроенный рекордер Telegram
- Для длинных сообщений разбейте на части
- Поддерживаются основные языки (русский, английский)

#### Для голосовых сообщений:
- Говорите четко и отчетливо
- Избегайте сильного фонового шума

## ⚙️ Конфигурация

### Переменные окружения (.env)

```env
# Telegram Bot Configuration
TELEGRAM_BOT_TOKEN=your_bot_token_here
ADMIN_IDS=123456789,987654321  # ID администраторов через запятую

# Database Configuration
DATABASE_URL=postgresql://username:password@localhost:5432/ai_transcriber
REDIS_URL=redis://localhost:6379/0

# AI Models Configuration
WHISPER_MODEL=base
WHISPER_DEVICE=cpu  # cpu/cuda

# Application Settings
LOG_LEVEL=INFO
MAX_IMAGE_SIZE=20
TEMP_DIR=downloads
DEFAULT_LANGUAGE=ru

# Web Interface
WEB_HOST=0.0.0.0
WEB_PORT=5000
SECRET_KEY=your_secret_key_here

# Security
CORS_ORIGINS=http://localhost:3000,http://localhost:5000
```

### Доступные модели Whisper

- `tiny` - Самая быстрая, базовое качество
- `base` - Сбалансированная скорость и качество (рекомендуется)
- `small` - Улучшенное качество
- `medium` - Высокое качество
- `large` - Максимальное качество, требует много ресурсов

## 🐛 Устранение непроблем

### Частые проблемы

1. **Ошибка импорта Whisper**
   - Убедитесь, что PyTorch установлен корректно
   - Проверьте совместимость версий

2. **Tesseract не найден**
   - Установите Tesseract OCR
   - Добавьте в PATH переменные окружения

3. **Бот не отвечает**
   - Проверьте токен в `.env`
   - Убедитесь, что бот запущен

4. **Плохое качество распознавания**
   - Улучшите качество изображения
   - Попробуйте другую модель Whisper

### Логирование

Бот ведет подробный лог в файле `logs/bot.log`. При возникновении проблем проверьте лог для детальной диагностики.
